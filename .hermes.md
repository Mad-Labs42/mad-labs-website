## Figma Policy

**Figma is reserved for explicit design-polish checkpoints only.**

- Figma may NOT be used for design work unless the user says exactly:
  **"FIGMA is a GO!"**
- No other phrase, assumption, or "it seems useful" reasoning authorizes Figma use.
- Figma is reserved for: high-value visual review, polish passes, selected-frame
  inspection, and later comparison work only.
- The repo and browser output remain the source of truth for all design decisions.
- Figma MCP is configured and can be tested for connectivity, but must not be used
  to inspect design files, extract design direction, or convert frames to code.

**Configured MCPs:**
- `figma` — remote Figma MCP (https://mcp.figma.com/mcp), OAuth pending
- `figma-desktop` — Figma Desktop MCP (http://host.docker.internal:3845/mcp),
  reachable from WSL when Figma Desktop is running on Windows
- `penpot-mcp` — Penpot design tool MCP (URL + user token), available on next session start
- `lighthouse-mcp` — Quality audit tool, available on next session start
- `icogenie-mcp` — Icon generation, available on next session start

## Development

### Quick Start

```bash
# From WSL project root
cd /mnt/d/Workspaces/mad-labs-website

# Install dependencies
pnpm install

# Development server
pnpm dev

# Validate
pnpm check      # astro check (type checking)
pnpm lint       # eslint
pnpm format:check  # prettier
pnpm build      # production build
pnpm ci         # full validation suite
```

### Running Hermes on this project

```bash
cd /mnt/d/Workspaces/mad-labs-website && \
hindsight-embed -p mad-labs-website daemon start && \
hermes -s mad-labs-brand,mad-labs-website
```
